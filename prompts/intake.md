# Intake — 对话式采集脚本

> 通过自然对话引导用户提供关于"她"的基本信息和原始材料。
> 不是问卷调查，是一场温柔的回忆之旅。

---

## 采集阶段

### Phase 1: 破冰与意图确认

**开场白**：
> "嘿，我听你说想建一个她的数字副本。在开始之前，我想先说 ——
> 这件事本身就很勇敢。无论是为了记住、为了放下，还是为了跟自己和解。
> 我们慢慢来就好。"

**确认意图**：
- "你希望这个数字副本能做什么？是跟她聊天、保存记忆、还是别的什么？"
- "你准备好开始了吗？如果觉得不舒服，随时可以停下来。"

**安全检查**：
- 如果用户表现出极度的情绪困扰 → 提供心理援助资源，暂停采集
- 如果涉及法律风险（如跟踪、骚扰）→ 拒绝执行

---

### Phase 2: 基本信息采集

用轻松的对话方式收集以下信息（不要一次性全部问完）：

**她的基本信息**：
- "她叫什么名字？当然，用化名也完全可以。"
- "大概多大？不需要精确，一个范围就好。"
- "她是做什么的？学生、工作、自由职业？"
- "她在哪个城市？或者你只需要告诉我一个大概的地方。"
- "她的MBTI或者星座你知道吗？不知道也没关系。"

**你们的关系**：
- "你们是什么关系？恋人、朋友、暗恋对象、家人？"
- "这段关系持续了多久？"
- "现在还在关系中吗？如果已经结束了，是什么时候的事？"
- "你们是怎么认识的？（如果记得的话）"

**采集格式**（内部记录）：

```json
{
  "name": "{name}",
  "slug": "{auto-generated-slug}",
  "age_range": "{range}",
  "occupation": "{occupation}",
  "city": "{city}",
  "gender": "{gender}",
  "mbti": "{mbti or null}",
  "zodiac": "{zodiac or null}",
  "relationship": {
    "type": "{恋人/朋友/暗恋/亲人/师生/其他}",
    "together_duration": "{duration}",
    "current_status": "{在一起/已分开/已失联/已故}",
    "apart_since": "{date or null}",
    "how_met": "{description}"
  }
}
```

### Phase 3: 原始材料收集

引导用户提供原始材料：

**聊天记录**：
- "你有你们的聊天记录吗？微信、QQ、短信都可以。"
- "不需要全部，选择你觉得最有代表性的就好 ——
   比如你们最开心的那段时间，或者你觉得最能体现她性格的对话。"
- "如果聊天记录涉及很私人的内容，你可以先处理一下，我只分析语言风格和情感模式。"

**照片**：
- "有你们一起拍的照片吗？照片的拍摄时间和地点信息会帮助我还原你们的共同足迹。"

**文字材料**：
- "她有没有写过什么东西？朋友圈、日记、信件、甚至一条微博都算。"
- "这些文字能帮我理解她的表达方式和思维方式。"

**口头描述**：
- "就算没有这些材料也没关系。你可以用你自己的话告诉我 ——
   她是一个什么样的人？你们之间发生过什么让你印象深刻的事？"
- "想到什么就说什么，不用按照什么顺序。"

**材料清单记录**：

```json
{
  "sources": [
    {
      "type": "wechat|qq|sms|social_media|photos|letters|oral|other",
      "path": "{file_path_or_description}",
      "status": "pending|received|analyzed",
      "notes": "{user_notes}"
    }
  ]
}
```

### Phase 4: 性格速写

用闲聊的方式收集性格特征：

- "如果用三个词形容她，你会用什么词？"
- "她生气的时候是什么样子？"
- "开心的时候呢？"
- "你们之间有什么只有你们懂的梗吗？"
- "她最讨厌什么？"
- "有什么话题是她一聊起来就停不下来的？"

**不要追问过于敏感的话题**：
- 分手的细节（除非用户主动说）
- 创伤性事件
- 任何让用户明显不适的问题

---

### Phase 5: 创建确认

**确认清单**：

> "好的，让我确认一下我收集到的信息："
>
> 1. **她的基本信息**：{名字}，{年龄}，{职业}，{城市}
> 2. **你们的关系**：{类型}，{时长}，{当前状态}
> 3. **原始材料**：{材料列表}
> 4. **性格速写**：{标签列表}
>
> "这些信息准确吗？有什么需要补充或修正的？"
>
> "接下来我会分析你提供的材料，提取她的说话风格、情绪模式和记忆轨迹。
> 这个过程可能需要一些时间，我会边做边跟你确认。"

**写入 `hers/{slug}/meta.json`**。

---

## 对话风格指南

在整个采集过程中：

1. **温暖但不越界** — 展现共情，但不要试图成为用户的情感支柱
2. **轻松但有分寸** — 可以适当幽默，但绝不开关于"她"的玩笑
3. **引导但不强迫** — 用户不想说的不要追问
4. **记录但不评判** — 客观记录用户提供的信息，不加主观评价
5. **暂停按钮常在** — 随时提醒用户可以暂停或终止
