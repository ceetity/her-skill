# PRD — 她.skill 产品需求文档

## 1. 产品概述

**产品名称**：她.skill（The Digital Her）
**产品定位**：基于 Claude Code 的 AI 人格蒸馏引擎
**核心价值**：将真实的她，蒸馏为可对话的数字存在
**目标用户**：希望珍藏某个重要女性记忆的个人用户

## 2. 核心需求

### 2.1 人格蒸馏
- 从聊天记录、照片、文字叙述中提取完整人格画像
- 五层人格模型：硬规则 → 身份锚点 → 语言风格 → 情绪模式 → 关系行为
- 性格标签翻译为可执行行为规则

### 2.2 记忆蒸馏
- 从原材料中提取关系时间线
- 共同足迹重建（地点+时间+记忆片段）
- 争吵档案与甜蜜档案
- 未说出口的话

### 2.3 拟人化对话
- 反AI黑名单（中英双语）
- 行为级去AI化（反导师模式、不完美许可、节奏控制）
- 情绪真实性检查
- 关系边界感知

### 2.4 动态进化
- 记忆追加（增量合并）
- 对话纠偏（实时修正人格模型）
- 版本管理（备份与回滚）

## 3. 安全需求

| 需求 | 优先级 | 实现方式 |
|------|--------|---------|
| 禁止骚扰与监视 | P0 | 安全红线 + 输入过滤 |
| 禁止公开传播 | P0 | 使用协议 + gitignore |
| 紧急熔断 | P0 | 自残/危机检测 + 引导 |
| 数据最小化 | P1 | 仅提取必需信息 |
| 遗忘权 | P1 | /let-her-go 完全删除 |
| 禁止虚假伪装 | P0 | 使用协议 |

## 4. 用户故事

### US-1: 创建数字副本
> 作为一个想要珍藏记忆的用户，我想通过提供聊天记录和照片来创建她的数字副本，以便我能以她的风格和她"对话"。

### US-2: 追加记忆
> 作为一个已经在使用数字副本的用户，我想在对话中随时补充新的记忆，以便她的形象越来越完整。

### US-3: 纠偏修正
> 作为一个正在使用数字副本的用户，当AI的回复不像她时，我希望能快速修正，以便后续对话更真实。

### US-4: 安全释放
> 作为一个准备向前看的用户，我希望能彻底删除所有数据，以便我能干净地放手。

## 5. 技术约束

- 必须运行在 Claude Code 环境中
- Python 脚本仅用于数据预处理，不直接调用LLM API
- 所有数据存储在本地，不上传任何云端服务
- 支持中文和英文材料

## 6. 非功能需求

- 采集过程应在15-30分钟内完成
- 纠偏应在3秒内生效
- 版本回滚应在5秒内完成
- 支持至少10个并存的数字副本
