# Merger — 增量合并逻辑

> 处理用户在对话中追加的新记忆。
> 不是替换，是增量合入现有记忆结构。

---

## 触发条件

用户说出以下类型的语句时触发合并：
- "其实还有一件事..."
- "补充一下，我们还..."
- "对了，我记得那次..."
- "我忘了说..."
- "之前没提到..."

---

## 合并策略

### 判断新信息的归属模块

| 信息类型 | 合并目标 | 处理方式 |
|---------|---------|---------|
| 新的时间线事件 | `时间线` 表格 | 按时间插入正确位置 |
| 新的地点 | `共同足迹` 表格 | 追加到表格末尾 |
| 新的记忆片段 | `关键记忆片段` | 追加为新片段 |
| 新的梗/称呼 | `专属暗号` | 追加到对应区域 |
| 新的争吵案例 | `争吵档案` | 追加或更新高频原因 |
| 新的温暖记忆 | `温暖档案` | 追加到对应区域 |
| 新的性格特征 | `persona.md` | 合入对应 Layer |
| 新的语言习惯 | `persona.md Layer 2` | 更新统计数据 |

### 合并规则

1. **不覆盖现有内容** — 新信息追加，不替换已有记忆
2. **时间排序** — 时间线事件插入正确的时间位置
3. **冲突处理** — 如果新信息与现有信息矛盾：
   - 以用户最新确认的为准
   - 在旧信息处标注 `[已修正，见 Correction #N]`
4. **去重** — 如果新信息与已有内容实质相同，不重复添加
5. **来源标注** — 标注信息来源为"对话补充"

### 执行步骤

1. 接收用户的新信息
2. 判断信息类型和归属模块
3. 在对话中向用户确认理解是否正确：
   > "我理解的是：{复述新信息}。这个理解对吗？"
4. 用户确认后，使用 Edit 工具将信息插入对应文件的对应位置
5. 在文件末尾的 Correction/Change Log 区域记录此次合并：
   ```
   Merge #1 [{日期}]
   内容：{新信息摘要}
   插入位置：{文件/区域}
   来源：对话补充
   ```
6. 重新运行 `skill_writer.py combine` 更新合成 SKILL.md

---

## 版本管理

每次合并前自动创建版本备份：
- 使用 `tools/version_manager.py backup`
- 用户可通过 `/her-rollback` 回滚到合并前状态
